11. Glossary — Every Term in One Sentence

A plain-English dictionary of every technical term used in this guide. If a word confused you anywhere above, find it here.

The system & its parts

ORAOffice of Research Administration — the Morgan State office that helps researchers win and manage grants.
FrontendThe web pages you see and click, running in your browser (built with React + Vite).
BackendThe server program in the middle that handles logins, saves data, and forwards questions to the AI (FastAPI).
ADK AgentThe separate AI service that talks to Gemini and searches the knowledge base (Google's Agent Development Kit).
ServiceOne independently-running program with its own job (ORA Navigator has three).
PortA numbered "door" on a computer that a program uses to receive network requests.
EndpointOne specific web address on the backend that does one job (e.g. /chat/stream).
APIApplication Programming Interface — the set of endpoints one program offers so another can ask it for data or actions.

The AI & how it answers

GeminiGoogle's large language model — the AI that actually writes the answer sentences (here, gemini-2.5-flash).
LLMLarge Language Model — an AI trained to read and write human language.
Knowledge Base (KB)The 382 approved ORA documents — the only facts the bot is allowed to use.
RAGRetrieval-Augmented Generation — look up real documents first, then write the answer using only those.
GroundingBacking an answer with real retrieved documents instead of the AI's own memory.
Grounding chunkOne retrieved passage of KB text that backed part of the answer; chunks become the "Sources."
HallucinationWhen an AI confidently states something that isn't true / isn't in its sources.
FaithfulnessAn answer that only states things the KB actually supports — no invented facts.
Semantic searchSearching by meaning, not exact keywords (so "extend my grant" finds "no-cost extension").
Vertex AI SearchGoogle's service that does that semantic search over the 382 documents.
DatastoreGoogle's indexed, searchable store of documents (oranavigator-kb-v8).
IndexingReading every document once, ahead of time, to build a fast lookup structure.
TF-IDFA simple no-AI scoring method that ranks documents by rare, on-topic word overlap with your question.
PrefetchPasting a few likely-relevant docs into the AI's instructions in advance, as a head start.
Refusal / deflectionWhen the bot honestly says it doesn't have the info and points you to ORA, instead of guessing.
SSE (Server-Sent Events)A way to stream an answer to the browser piece-by-piece so you see it appear gradually.

Memory & personalization

EmbeddingA list of numbers that captures the meaning of a piece of text; similar texts get similar numbers.
Cosine similarityA 0-to-1 score for how alike two embeddings are (1 = same meaning).
SessionOne continuous conversation (one chat window).
TurnOne question-and-answer pair.
ConsolidationDistilling recent chats into a few durable one-line facts (and capping them), without deleting the chats.
Profile mirroringCopying your profile fields (dept/role/interests) into the memory the AI reads.
Prompt injectionA hidden instruction smuggled into text to trick the AI into disobeying its rules; the memory "NOT instructions" label defends against it.

Data, storage & security

Database / table / column / rowAn organized store of information; a table is one "spreadsheet" with named columns and one record per row.
Cloud SQLGoogle's managed MySQL database service (instance oranavigator-db).
MySQLA popular relational database program.
SQLAlchemy / ORMA Python tool that lets the backend describe and use database tables as Python classes.
JSONA plain-text way of writing labeled information (like a form with named boxes).
CacheA short-term memory of recent results so the app doesn't redo expensive work.
RedisA separate fast in-memory database used here purely for caching (the "L2" layer).
TTLTime To Live — how long a cached entry stays before it expires.
SSL / TLSEncryption that scrambles network traffic so nobody can read it in transit.
JWTJSON Web Token — the signed "wristband" that proves you're logged in.
bcryptA deliberately slow one-way scrambler for passwords, so the real password is never stored.
Authentication / AuthorizationProving who you are / deciding what you're allowed to do.
Secret ManagerGoogle's locked vault for passwords and keys, kept out of the code.
IdempotencyA guarantee that doing something twice has the same effect as doing it once (e.g. no duplicate reminder emails).

Cloud & operations

GCPGoogle Cloud Platform — Google's rented-computing service where the live app runs.
Cloud RunGoogle's service for running a containerized program without managing servers.
Container / imageA self-contained package of a program plus everything it needs to run.
Artifact RegistryThe private shelf where built container images are stored.
Service accountA robot identity a program logs in as, so it can be granted permissions.
IAM / roleIdentity and Access Management — Google's permission system; a "role" is a bundle of permissions.
Cron jobA task that runs automatically on a fixed schedule.
Cloud SchedulerGoogle's "alarm clock" that triggers cron jobs at set times.
DeployPublishing the latest code from a laptop to the live Cloud Run services.
PWAProgressive Web App — a website that installs like an app and caches itself with a "service worker."
Service workerThe background script that caches the app's files (why you check changes in incognito).

Grants & compliance (the subject matter)

PIPrincipal Investigator — the lead researcher responsible for a grant.
Pre-award / Post-awardEverything before winning a grant (writing/submitting) / after winning it (spending/reporting).
Solicitation / FOAThe sponsor's announcement PDF describing a funding opportunity and its rules.
F&A rateFacilities & Administrative (indirect cost) rate — the overhead percentage a university charges on a grant.
IRBInstitutional Review Board — approves research involving human subjects.
IACUCInstitutional Animal Care and Use Committee — approves research involving animals.
SOPStandard Operating Procedure — a numbered procedure document (note: IACUC SOP 37 doesn't exist).
COIConflict of Interest — rules about disclosing money/relationships that could bias research.
RCRResponsible Conduct of Research — required research-ethics training.
No-cost extension (NCE)Extra time to finish a grant past its end date, with no extra money.
Biosketch / DMPA short researcher CV / a Data Management Plan — common required proposal attachments.
